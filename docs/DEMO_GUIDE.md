# BlinkShield AI Demo Guide

## 2-Minute Phase 2 Demo

1. Open the worker login and sign in with `asha@blinkshield.demo / Pass@123`.
2. Show the worker dashboard with weekly pricing and active weekly policy.
3. Open the live monitor page and explain that BlinkShield watches Blinkit dark-store zones instead of generic city-wide risk.
4. Switch to the admin login with `admin@blinkshield.demo / Admin@123`.
5. Use Demo Mode to simulate `Heavy Rainfall` in `Koramangala-5th-Block`.
6. Refresh the worker claims page to show the auto-triggered claim and payout.
7. Open the admin fraud page and explain that suspicious claims are flagged, not auto-rejected.

## 5-Minute Final Demo

1. Introduce the problem: Blinkit partners lose 20-30% weekly income during rain, heat, AQI spikes, closures, and local outages.
2. Walk the landing page and highlight the Blinkit-specific positioning:
   - dark-store based
   - hyperlocal zones
   - weekly pricing
   - income protection only
3. Show onboarding:
   - delivery partner enters work pattern
   - risk agent generates a personalized weekly quote
   - explainability panel shows premium drivers
4. Show zero-touch flow:
   - admin simulates severe pollution or zone restriction
   - disruption gets stored
   - fraud agent checks worker telemetry
   - claim agent decides
   - payout agent returns a mock payout success
5. Show seeded historical claims and payouts to prove the system is already populated.
6. Show fraud fairness:
   - run `GPS Spoof Attack`
   - point to the fraud alerts page
   - explain that BlinkShield flags anomalies instead of blanket rejection

## Recommended Talking Points

- Weekly pricing matches gig earnings cycles.
- Coverage is only for loss of income.
- Dark-store closures and micro-zone outages matter more than city-wide averages for Blinkit.
- Explainability makes premium and claim outcomes demo-friendly and trustworthy.
- Mock mode keeps the project fully runnable without paid API access.
